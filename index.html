Safe Share PDF

공유 전 PDF
민감정보 제거

SSP는 PDF 문서의
개인정보, 업무기밀 숨겨진 메타데이터를
AI로 탐지하고 안전한 공유용 PDF를
생성하는 보안 시스템입니다.

주요 기능 보기 시연 확인

The Problem

보이지 않는다고 삭제된 것은 아닙니다.

검은 박스, 단순 가림, 화면상 마스킹만으로는 PDF 내부 텍스트와 메타데이터가 그대로 남을 수 있습니다.

마스킹 처리의 허점

복사와 붙여넣기 과정에서
원본 텍스트가 다시 노출될 수 있습니다.

숨겨진 데이터

Metadata와 Hidden text 레이어가
문서 안에 남아 보안 위험을 만듭니다.

실제 사고 사례

PDF 내부 구조에 대한 이해 부족은
실제 기밀 정보 유출로 이어졌습니다.

Real Cases

이미 실제로 일어난 문제입니다.

미국 Paul Manafort 사건

법원 제출 PDF 문서를 검은 박스로 처리한 뒤 공개했지만
Copy & Paste만으로 원문이 그대로 노출되었습니다.

시각적 마스킹만으로는
데이터를 보호할 수 없다는 대표 사례입니다.

영국 국방부 사례

핵잠수함 관련 PDF 문서를 공개하면서
텍스트 레이어가 제거되지 않아 기밀 정보가 노출되었습니다.

PDF 내부 구조에 대한 이해 부족이
보안 사고로 이어진 사례입니다.

삼성전자 AI 오남용 사건

DS 부문에서 챗GPT에 내부 정보와
설비 계측 소스 코드, 회의 내용이 입력되었습니다.

클라우드 AI 사용 시
민감 문서 자체가 외부 서버로 전송될 수 있습니다.

Current Security Limits

기존 보안은 문서 내부를 끝까지 보지 못합니다.

DRM

접근 권한 제어 중심 구조라
화면 촬영 등 우회가 가능하고 대응이 제한됩니다.

DLP

규칙 기반 탐지 방식으로 운영되어
문맥 분석과 비정형 데이터 탐지가 어렵습니다.

Redaction

화면상 마스킹 중심 처리로
OCR 레이어와 Metadata 복구 위험이 남을 수 있습니다.

How SSP Works

찾아내고 지우고 새로 만듭니다

SSP는 단순히 덮어씌우지 않습니다. 민감정보를 식별하고 실제 데이터를 제거한 뒤, 공유 가능한 새 PDF를 만듭니다.

Core Features

한국어 문서 환경에 맞춘 보안 처리

SSP는 국내 기업 및 행정 문서의 한글, 영문, 숫자 혼합 구조를 반영하고, 사용자 검토를 거쳐 유지, 마스킹, 삭제 대상을 결정합니다.

NER 기반 개체 인식

KoELECTRA 기반 NER로 Entity를 분류하고
주요 민감정보를 탐지합니다.

위험도 점수 산정

정보 유형과 조합을 기반으로 위험도를 계산해
우선 검토할 항목을 제시합니다.

사용자 최종 검토

자동 탐지 결과를 UI에서 확인하고
유지, 마스킹, 삭제 여부를 최종 결정합니다.

Example

문장에서 실제 마스킹 대상을 찾아냅니다.

NER 기반 식별 결과

예시 문장: 장미 식당에서 2만원 결제했습니다.

  • 장미식당 → [가맹점명], 신뢰도 0.529
  • 2만원 → [결제금액], 신뢰도 0.885
  • 마스킹 텍스트: [가맹점명]에서 [결제금액] 결제했습니다.

Secure PDF Generation

단순 오버레이가 아니라 실제 보안 처리입니다.

원문 제거, 길이 보존형 마스킹, 문자열 치환을 함께 수행해 복구 가능한 취약점을 줄입니다.

AI Model

문장 맥락과 개체 정보를 함께 봅니다.

Sentence Classifier가 위험 문장을 먼저 선별하고, NER 모델이 실제로 마스킹할 단어를 찾아냅니다.

Sentence Classifier

계약금, 계좌, 내부문서, 보안정보 등
민감정보 가능성이 높은 문장을 우선 탐지합니다.

NER 모델

이름, 전화번호, 주소, 계약번호 등
실제 마스킹할 텍스트를 유형별로 분류합니다.

Differentiation

기존 프로그램과 다르게, 문맥과 잔존 데이터를 함께 봅니다.

메타데이터까지 완전 삭제

문서의 메타데이터와 민감정보를 함께 제거해
잔존 가능성을 최소화합니다.

문맥·의미 기반 탐지

단순 키워드 검색이 아니라
문장의 의미와 맥락을 분석합니다.

자동 탐지를 통한 편의성

AI가 자동으로 후보를 제시해
검토 시간을 줄이고 업무 효율을 높입니다.

Limitations

한계도 분명히 인식합니다.

학습용 데이터 부족

계약서, 업무 문서, 내부 보고서는
기관과 기업마다 표현 방식이 다릅니다.

특정 기업의 내부 용어와 계약 번호 체계는
탐지 성능이 낮을 수 있습니다.

PDF 품질 차이에 따른 텍스트 추출 편차

PDF마다 텍스트 레이어와 표 구조가 달라
같은 문장도 다르게 추출될 수 있습니다.

입력 단위가 달라지면
Sentence Classifier와 NER 결과에도 영향을 줍니다.

Impact

공유 전 보안 점검을 누구나 쉽게

보안 측면 고위험 개인정보와 업무기밀의
외부 유출 위험도를 낮춥니다.
업무 측면 비전문 사용자도 공유 전 보안 점검과
검토 우선순위 확인이 가능합니다.
비즈니스 측면 최소 공개 원칙을 적용해
사후 대응 비용과 검토 부담을 줄입니다.
